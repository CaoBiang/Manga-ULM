# 规则

## 0. 核心规则
- 禁止以打补丁的方式修改代码，任何问题都要找到根因、任何改动都要修改根源。
- 禁止使用 `git` 命令。

## 1. 项目定位

- **核心定位**：本工程是一个本地漫画管理器 + 漫画阅读器。
- **开发阶段**：不要在当前的架构上打补丁，无需考虑向前兼容，永远要大刀阔斧地重构。
- **编码准则**：高质量、高可用、高稳定、结构清晰、代码优雅、解耦。

## 2. 文档规范

- **存放目录**：`\docs` 文件夹中存放文档，其下包含三个文件夹，每个文件夹下根据整体、各个模块进行划分描述，三个文件夹分别为：
  - `\design`：描述设计目的、整体设计、模块介绍。
  - `\use`：描述使用场景、使用方法。
  - `\standard`：描述针对模块的详细编码规范，尤其包含复用规范。
- **同步维护**：任何功能修改、新增需同步更新 MD 文档及 Mermaid 图表。
- **内容要求**：采用总分方法，简洁易懂、面向小白、多多举例、尽量减少大段代码。
- **Mermaid**：内容需使用双引号括住。

## 3. 编码规则

- **语言编码**：强制使用 **UTF-8** 编码。
- **中文优先**：代码注释、文档、输出内容全程使用中文。

## 4. 前端规范

- **技术栈**: 使用 Vue + AntDesign + i18n。
- **组件与风格**: 漫画阅读器中使用统一的自定义组件、样式，以黑白灰色毛玻璃为主。漫画管理器中尽可能参考、使用漫画阅读器中的毛玻璃的样式、模糊效果，最好基于 AntDesign-Vue 的组件体系做毛玻璃化，但允许更丰富的颜色，禁止使用渐变色、高对比度、高饱和度。

## 5. 其他要求
- 新增任何功能时、针对所有可能的配置项，都不能写死，必须允许用户自定义，放在设置页面中。
- 设置部分保证结构合理、清晰易懂，有需要单独摘出来的设置模块就要在侧边栏的“设置”中新增一个模块。
- 所有前端都要使用 i18n 进行多语言支持。
- 确保前端视觉统一，尽可能复用组件，复用 Vue 组件按照合理的结构按模块统一划分，所有复用组件的文档都要写在 `standard` 对应模块中。
