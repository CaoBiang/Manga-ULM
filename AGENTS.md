# 规则

## 0. 核心规则
- 禁止以打补丁的方式修改代码，任何问题都要找到根因、任何改动都要修改根源。
- 请使用陈述性语言描述现状（直接描述法），避免使用对比性措辞。
- 确保开箱即用。

## 1. 项目定位

- **核心定位**：本工程是一个本地漫画管理、漫画阅读器。
- **开发阶段**：不要在当前的架构上打补丁，无需考虑向前兼容，永远要大刀阔斧地重构。
- **编码准则**：高质量、高可用、高稳定、结构清晰、代码优雅、解耦。

## 2. 文档规范

- **同步维护**：代码修改需同步更新 MD 文档及 Mermaid 图表。
- **内容要求**：简洁易懂、面向小白、尽量减少大段代码。至少包含设计目的、整体设计、模块介绍、使用场景、使用方法。
- **文档分层**：有一个 `README.md` 入口文档，以及以每个子模块命名的子文档。
- **Mermaid**：内容需使用双引号括住。

## 3. 结构与依赖

- **目录分工**：`\docs` 文件夹中存放文档。
- **联动修改**：修改代码时，必须同步更新相关联的文档，保持接口与说明一致。

## 4. 编码规范

- **语言编码**：强制使用 **UTF-8** 编码。
- **中文优先**：代码注释、文档、输出内容全程使用中文。
- **日志规范**：必须使用 `loguru.logger`。

## 5. 前端规范

- **技术栈**: 使用 Vue + AntDesign 。
- **组件规范**: 使用 AntDesign 组件。
- **风格要求**：符合 AntDesign 样式、简洁清晰，颜色禁止使用渐变色、高对比度、高饱和度。
